# Описание хода выполнения работы

Вариант для выполнения предполагает написать программу для работы со временем
суток. Основная единица данных — момент времени, представленный часами, минутами
и секундами. Программа реализует две операции: вычитание заданного числа секунд
из момента времени и подсчёт количества секунд между двумя моментами времени
в пределах одних суток.

В среде разработки создадим консольный проект. Программа организована в пяти
файлах: заголовочные файлы timer.h и group.h содержат объявления классов,
файлы реализации timer.cpp и group.cpp — определения методов, main.cpp —
логику меню и взаимодействия с пользователем.

Опишем в файле timer.h класс Time, содержащий три закрытых поля: hours
(часы), minutes (минуты) и seconds (секунды). Такой подход иллюстрирует
принцип инкапсуляции: внутреннее состояние объекта скрыто от внешнего кода,
а доступ к нему предоставляется только через методы класса.

Напишем четыре конструктора: конструктор по умолчанию, инициализирующий
время значением 00:00:00; конструктор с параметрами, принимающий часы, минуты
и секунды; конструктор из общего числа секунд, позволяющий создать объект
из единственного числового значения; конструктор копирования. Также определим
деструктор.

Для доступа к полям реализуем геттеры getHours, getMinutes, getSeconds
и сеттеры setHours, setMinutes, setSeconds, setTime с вызовом нормализации
после каждого изменения. Добавим методы преобразования: toSeconds переводит
момент времени в общее количество секунд от начала суток, fromSeconds
восстанавливает время из такого значения.

Центральными для задания являются два метода. Метод deductSeconds вычитает
заданное количество секунд из текущего момента времени: он переводит время
в секунды, выполняет вычитание и восстанавливает результат через fromSeconds.
Метод secondsBetween возвращает абсолютную разницу в секундах между двумя
моментами времени — оба момента переводятся в секунды, разность берётся
через abs(), поэтому порядок передачи аргументов не важен.

Для корректной работы арифметических операций реализуем закрытый вспомогательный
метод normalize, который приводит значения полей к допустимым диапазонам.
Например, если после вычитания секунды стали отрицательными, метод выполняет
перенос в минуты, минуты — в часы, а при выходе за границы суток значение
оборачивается по модулю 24 часа.

Добавим метод inputFromConsole для ввода времени с клавиатуры. Каждое поле
запрашивается отдельно с проверкой допустимого диапазона: часы от 0 до 23,
минуты и секунды от 0 до 59. При некорректном вводе или нечисловом символе
программа выводит сообщение об ошибке и повторно запрашивает значение.
Статический метод isValid выполняет проверку трёх значений одновременно.

Для удобного форматирования вывода реализуем метод toString, возвращающий
строку в формате ЧЧ:ММ:СС с ведущими нулями. Перегрузим оператор вывода <<,
который использует toString внутри, и оператор ввода >> для чтения времени
в формате ЧЧ:ММ:СС из потока.

Перегрузим арифметические операторы: оператор - возвращает новый объект Time
с вычтенными секундами, оператор -= изменяет текущий объект и возвращает
ссылку на него. Реализуем операторы сравнения ==, < и >, а также оператор
присваивания =.

Создадим класс Group в файлах group.h и group.cpp. Класс представляет
динамический массив объектов Time и содержит два закрытых поля: указатель
timers на массив и size — его размер. Конструктор Group(int size) выделяет
память под массив объектов через оператор new[]. Поскольку класс управляет
динамической памятью, реализуем конструктор копирования и оператор =
с глубоким копированием: при копировании объекта Group создаётся новый
независимый массив. Без этого два объекта указывали бы на одну область
памяти, что привело бы к двойному освобождению при вызове деструктора.
Деструктор освобождает память через delete[]. Добавим методы setTimer
для установки объекта по индексу, substractSeconds для вычитания секунд
из элемента по индексу, getSize для получения размера массива, а также
оператор [] для доступа к элементам.

В файле main.cpp реализуем консольное меню. При запуске программа
отображает список из четырёх пунктов: вычитание секунд, подсчёт секунд
между двумя моментами, демонстрация всех операций, выход. Меню работает
в цикле do-while: после выполнения любого пункта меню отображается повторно,
пока пользователь не выберет пункт 0.

Каждый пункт меню реализован в отдельной функции. Функция
subtractSecondsOperation создаёт объект Time, вызывает inputFromConsole,
запрашивает количество секунд для вычитания и применяет перегруженный
оператор -. Функция calculateSecondsBetween создаёт два объекта Time,
запрашивает их значения и выводит результат secondsBetween в секундах
и в виде часов, минут, секунд. Функция demonstrateAllOperations
последовательно демонстрирует все конструкторы, операторы, методы сравнения
и класс Group.

Для наблюдения за жизненным циклом объектов введём глобальный флаг
g_verbose. Конструкторы и деструктор класса Time выводят диагностические
сообщения только при g_verbose == true. Флаг включается в начале функции
demonstrateAllOperations и выключается по её завершении, поэтому
диагностический вывод появляется исключительно в режиме демонстрации
и не мешает работе основных операций.
